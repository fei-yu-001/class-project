# 工资管理系统 - 开发日志

## 2026-06-28 全面业务优化迭代

### 背景
对课程设计级别的工资管理系统进行企业级业务逻辑优化，使其从"CRUD 演示"升级为"可演示的企业原型"。

### 优化范围（7 大维度，30+ 文件改动）

---

#### 一、薪资计算引擎重构

**问题**: 原系统个税和社保写死为 0，加班费统一 1.5 倍，所有业务常量硬编码。

**改动**:
- 新增 `TaxCalculator`：中国个税 7 档累进税率计算（起征点 5000，3%~45%）
- 新增 `InsuranceCalculator`：五险一金个人部分（养老 8% + 医疗 2% + 失业 0.5% + 公积金 12%）
- `SalaryServiceImpl.calculate()` 重构：加班费按 `otType` 区分 1.5x/2x/3x，所有常量从 `sys_config` 表读取
- `SalaryCalculationPreview` 新增 `taxDeduction`/`insuranceDeduction` 字段

**验证**: 10000 元基本工资、A 级绩效、3 小时工作日加班 → 预览结果应含个税和社保扣款。

---

#### 二、系统配置化

**问题**: 21.75 工作日、300 元全勤奖、50 元迟到扣款等全部硬编码，改参数需改代码。

**改动**:
- 新建 `sys_config` 表（12 项配置：税率阈值、社保费率、加班倍率、全勤奖金额等）
- 新建 `SysConfigService`/`SysConfigController` → `GET/PUT /api/sys-config`
- 新建 `SysConfigView.vue`：管理员可在页面上直接修改配置值

---

#### 三、请假/加班审批流

**问题**: 请假和加班创建即 APPROVED，没有审批流程。前端 API 有 approve/reject 端点但从未调用。

**改动**:
- `LeaveRequest`/`OvertimeRecord` 默认状态从 `APPROVED` → `PENDING`
- `AttendanceServiceImpl` 新增 4 个审批方法（带状态转换校验：只有 PENDING 可审批）
- `AttendanceController` 新增 4 个端点：`PUT /attendance/leaves/{id}/approve|reject`、`PUT /attendance/overtime/{id}/approve|reject`
- `LeaveView.vue`/`OvertimeView.vue` 新增"通过"和"驳回"行按钮（仅 PENDING 状态显示）
- `OvertimeRecord` 新增 `otType` 字段（WEEKDAY/WEEKEND/HOLIDAY）

---

#### 四、权限与安全加固

**问题**: 薪资读取端点无权限控制，API 限流是死代码，XSS 过滤不覆盖 JSON body，LoginRequest 无长度限制。

**改动**:
- `SalaryController`：`getById`/`getByEmployee`/`search` 添加 `@RequireRole("ADMIN")`
- 新建 `RateLimitFilter`：基于 Redis 的 IP 限流（登录 10 次/分，API 120 次/分）
- `XssFilter` 增强：缓存请求体，逐字符解析 JSON 结构，仅对字符串值做 XSS 消毒
- `LoginRequest` 添加 `@Size(max=50)` 和 `@Size(max=100)` 约束
- 考勤表添加 `uk_att_emp_date` 唯一约束，后端同步校验

---

#### 五、审计日志

**问题**: 谁在什么时候做了什么操作完全没有记录。

**改动**:
- 新建 `audit_log` 表（userId, username, action, entityType, entityId, detail, createdAt）
- 新建 `AuditAspect` AOP 切面：拦截所有 `@RequireRole` 方法，自动记录操作
- `AuditLogServiceImpl.log()` 使用 `@Async` 异步写入，不影响主流程性能
- 新建 `AuditLogView.vue`：分页查看操作日志

---

#### 六、角色差异化

**问题**: 管理员和普通用户看到完全相同的仪表盘（全局统计数据）。

**改动**:
- `DashboardServiceImpl.getStats()` 根据 `SecurityContextHolder` 角色分流
- ADMIN → `getAdminStats()`：全局统计 + 3 个 ECharts 图表（带 Redis 缓存）
- USER → `getUserStats()`：个人数据（姓名、基本工资、最近实发、绩效等级、奖金/扣除明细）
- `DashboardView.vue`：用 `<template v-if="isAdmin">` 切换两种完全不同的界面

---

#### 七、假期余额 + 批量操作 + 部署

**改动**:
- 新建 `leave_balance` 表，预设每人 15 天年假
- 审批请假时自动扣除余额，余额不足拒绝
- 新增 `batchApprove`/`batchPay` 端点和前端批量按钮
- 生成工资按钮（之前 `handleGenerate` 函数存在但无 UI 触发）
- `WebMvcConfig` 添加 SPA 路由兜底
- 更新 `static/` 为最新前端构建
- 新增 `deploy.bat` 一键部署脚本
