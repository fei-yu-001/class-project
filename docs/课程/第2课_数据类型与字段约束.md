# 第2课：数据类型与字段约束

---

## 一、为什么需要数据类型？

想象你在填一张表格：
- "姓名"那一栏 → 你会填文字
- "年龄"那一栏 → 你会填数字
- "出生日期"那一栏 → 你会填日期

数据库也一样，每个字段都要规定**能存什么类型的数据**。这样做有两个好处：
1. **防止填错**：年龄栏填了"张三"，数据库会拒绝
2. **节省空间**：数字比文字占的空间小得多

## 二、你的项目中用到的数据类型

### 2.1 VARCHAR(n) —— 可变长度字符串

```sql
dept_name VARCHAR(50) NOT NULL COMMENT '部门名称'
```

| 要点 | 说明 |
|------|------|
| 是什么 | 存文字（字符串），长度可以变化 |
| n 的含义 | **最多**能存多少个字符 |
| 实际占用 | 按实际长度存，不会浪费空间 |

**举例**：
- `VARCHAR(50)` 最多存50个字符
- 你存了"技术部"（3个字），实际只占3个字符的空间，不会占50个

> **常见疑问**：VARCHAR(10) 能存几个汉字？
> 答：在 utf8mb4 字符集下，**10个汉字**。VARCHAR的数字是字符数，不是字节数。

**你项目中的使用**：
| 字段 | 类型 | 含义 |
|------|------|------|
| dept_code | VARCHAR(10) | 部门代码，如 "D001" |
| dept_name | VARCHAR(50) | 部门名称，如 "技术部" |
| emp_name | VARCHAR(30) | 员工姓名，如 "张三" |
| emp_status | VARCHAR(12) | 在职状态，如 "在职"、"离职" |

---

### 2.2 CHAR(n) —— 固定长度字符串

```sql
gender CHAR(1) NOT NULL COMMENT '性别'
```

| 要点 | 说明 |
|------|------|
| 是什么 | 存文字，但长度固定 |
| n 的含义 | 必须存满n个字符 |
| 实际占用 | 不管你存多少，都占n个字符的空间 |

**举例**：
- `CHAR(1)` 存 "男" 或 "女"，刚好1个字符
- 如果用 `CHAR(10)` 存 "男"，后面会自动补9个空格，凑满10个

> **VARCHAR vs CHAR 怎么选？**
> - 长度经常变化的 → 用 VARCHAR（如姓名、地址）
> - 长度总是固定的 → 用 CHAR（如性别只有1个字、手机号总是11位）
> - CHAR 稍微快一点，但差距很小，大多数场景用 VARCHAR 就行

### 2.3 INT —— 整数

```sql
emp_id INT NOT NULL AUTO_INCREMENT COMMENT '工号'
```

| 要点 | 说明 |
|------|------|
| 是什么 | 存整数（没有小数点） |
| 范围 | 大约 -21亿 到 +21亿 |

**你项目中的使用**：
| 字段 | 含义 |
|------|------|
| emp_id | 工号，如 1001 |
| proj_id | 项目ID |
| review_id | 考核ID |

---

### 2.4 DECIMAL(p, s) —— 精确小数

```sql
base_salary DECIMAL(10,2) NOT NULL COMMENT '基本工资'
```

| 参数 | 含义 | 举例 |
|------|------|------|
| p (precision) | 总共多少位数字 | 10 = 最多10位 |
| s (scale) | 其中小数点后几位 | 2 = 小数点后2位 |

**所以 `DECIMAL(10,2)` 的意思是**：
- 总共最多10位数字
- 其中2位是小数
- 整数部分最多8位
- 范围：-99999999.99 到 99999999.99

> **⚠️ 重点警告：为什么不用 FLOAT 或 DOUBLE？**
> FLOAT 和 DOUBLE 是浮点数，会有精度丢失的问题：
> ```
> 0.1 + 0.2 = 0.30000000000000004  ← 浮点数的坑！
> ```
>
> **🔥 铁律：涉及钱的地方，必须用 DECIMAL，保证一分钱都不差！**
>
> 这不是建议，是必须！用FLOAT存金额，是数据库设计中最常见的严重错误之一。
> 钱算错了，那就是真金白银的损失！

**你项目中的使用**：
| 字段 | 类型 | 含义 |
|------|------|------|
| base_salary | DECIMAL(10,2) | 基本工资，如 15000.00 |
| gross_total | DECIMAL(10,2) | 应发合计 |
| net_pay | DECIMAL(10,2) | 实发工资 |
| contrib_coeff | DECIMAL(3,2) | 贡献系数，如 1.00（最多3位，2位小数） |
| ot_hours | DECIMAL(4,1) | 加班时长，如 2.5（最多4位，1位小数） |

### 2.5 DATE —— 日期

```sql

hire_date DATE NOT NULL COMMENT '入职日期'
```

| 要点 | 说明 |
|------|------|
| 是什么 | 存日期，不含时间 |
| 格式 | 'YYYY-MM-DD' |
| 举例 | '1995-06-15' |

## 四、字段约束

约束就是给字段加**规则**，限制什么数据可以存进去。

### 4.1 NOT NULL —— 不能为空

```sql
dept_code VARCHAR(10) NOT NULL COMMENT '部门代码'
```

| 要点 | 说明 |
|------|------|
| 含义 | 这个字段**必须填**，不能留空 |
| 不加 NOT NULL | 字段可以不填（存 NULL） |

**类比**：填表格时，带星号 * 的必填项就是 NOT NULL。

**你项目中的规则**：
- 员工姓名 → NOT NULL（必须有名字）
- 上级部门代码 → 没有NOT NULL（顶级部门没有上级，可以为空）

---

### 4.2 DEFAULT —— 默认值

```sql
parent_dept_code VARCHAR(10) DEFAULT NULL COMMENT '上级部门代码'
contrib_coeff DECIMAL(3,2) NOT NULL DEFAULT 1.00 COMMENT '贡献系数'
```

| 要点 | 说明 |
|------|------|
| 含义 | 如果不填这个字段，就自动用默认值 |
| DEFAULT NULL | 不填就是空 |
| DEFAULT 1.00 | 不填就是1.00 |

**🔥 重点理解 `NOT NULL DEFAULT 1.00`**：

这句话拆开来读：
- `NOT NULL` → 这个字段**不能为空**（必须有值）
- `DEFAULT 1.00` → 如果你不填，**自动帮你填1.00**

所以效果是：**你填了就用你填的值，你不填就自动是1.00，但绝不会是空（NULL）**。

> **类比**：就像点外卖，口味默认是"微辣"（DEFAULT 1.00），你可以改成"中辣"或"不辣"，但你不能不选（NOT NULL）。

**举例**：
- 新增一个项目成员，不填贡献系数 → 自动填 1.00
- 新增一个项目成员，填了贡献系数 1.50 → 用你填的 1.50
- 新增一个部门，不填上级部门 → 自动填 NULL（空）

---

### 4.3 COMMENT —— 注释

```sql
emp_name VARCHAR(30) NOT NULL COMMENT '姓名'
```

| 要点 | 说明 |
|------|------|
| 含义 | 给字段加说明文字 |
| 作用 | 帮助别人理解这个字段是什么意思 |
| 不影响功能 | 注释只是说明，不影响数据存储 |

> **COMMENT 非常重要！** 几个月后你自己都忘了这个字段是什么意思，注释就是你的"说明书"。

## 五、约束组合实战

看看你项目中的几个典型组合：

```sql
-- 必填，没有默认值 → 插入数据时必须手动填
dept_name VARCHAR(50) NOT NULL COMMENT '部门名称'

-- 必填，有默认值 → 不填就用默认值
contrib_coeff DECIMAL(3,2) NOT NULL DEFAULT 1.00 COMMENT '贡献系数'

-- 非必填，默认为空 → 可以不填
parent_dept_code VARCHAR(10) DEFAULT NULL COMMENT '上级部门代码'

-- 非必填，也没默认值 → 不填就是NULL
card_type VARCHAR(20) DEFAULT NULL COMMENT '卡类型'
```

**总结规律**：

| 组合 | 含义 | 类比 |
|------|------|------|
| NOT NULL（无DEFAULT） | 必须手动填 | 必填项 |
| NOT NULL + DEFAULT | 可以不填，自动用默认值 | 有默认值的必填项 |
| DEFAULT NULL | 可填可不填 | 选填项 |

## 六、本课小结

```
数据类型
├── VARCHAR(n)  → 可变长文字（姓名、地址）
├── CHAR(n)     → 固定长文字（性别、编号）
├── INT         → 整数（ID、数量）
├── DECIMAL(p,s)→ 精确小数（金额必须用它！）
└── DATE        → 日期（出生日期、入职日期）

字段约束
├── NOT NULL    → 必填
├── DEFAULT     → 默认值
└── COMMENT     → 注释说明
```

**核心记忆**：
1. 🔥 涉及金额 → **必须用 DECIMAL**，不能用 FLOAT，这是铁律！
2. NOT NULL = 必填，DEFAULT = 默认值，NOT NULL DEFAULT = 必填但有默认值兜底
3. COMMENT 是给自己和他人看的说明书，一定要写
