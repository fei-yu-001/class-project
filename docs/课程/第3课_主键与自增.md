# 第3课：主键 PRIMARY KEY 与自增 AUTO_INCREMENT

---

## 一、什么是主键？

想象一个班级里的学生：
- 姓名可能重复（两个"张三"）
- 年龄肯定重复（很多人同岁）
- 但**学号**绝对不会重复

**主键 = 学号**，就是用来**唯一标识**每一行数据的那一列（或几列）。

### 主键的两大规则

| 规则 | 含义 | 举例 |
|------|------|------|
| **唯一性** | 每一行的主键值不能重复 | 工号1001只能出现一次 |
| **非空性** | 主键不能为空（NOT NULL） | 工号不能不填 |

> **一句话记忆**：主键就是每一行的"身份证号"，不能重复，不能为空。

---

## 二、你项目中的主键

### 2.1 单列主键（最常见）

```sql
-- 部门表：部门代码是主键
PRIMARY KEY(dept_code)

-- 员工表：工号是主键
PRIMARY KEY(emp_id)

-- 项目表：项目ID是主键
PRIMARY KEY(proj_id)
```

这些都是**一个字段**做主键，最简单也最常见。

### 2.2 联合主键（复合主键）

```sql
-- 项目成员中间表：员工工号+项目ID 一起做主键
PRIMARY KEY(emp_id, proj_id)

-- 工资-奖金关联表：工资记录ID+奖金ID 一起做主键
PRIMARY KEY(salary_id, bonus_id)
```

**为什么要用联合主键？**

以项目成员表为例：
- 员工1001可以参与多个项目 → emp_id 会重复
- 项目2001可以有多个成员 → proj_id 会重复
- 但"员工1001参与项目2001"这个组合 → **只会出现一次**

| emp_id | proj_id | 合法？ | 原因 |
|--------|---------|--------|------|
| 1001 | 2001 | ✅ | 第一次出现 |
| 1001 | 2002 | ✅ | 同一人不同项目 |
| 1002 | 2001 | ✅ | 同一项目不同人 |
| 1001 | 2001 | ❌ | 重复了！联合主键(1001,2001)已存在 |

> **记忆口诀**：单列主键像身份证号，联合主键像"姓名+出生日期"的组合。

## 三、AUTO_INCREMENT —— 自增

### 3.1 它解决什么问题？

假设你要新增一个员工，工号怎么填？
- 手动填？你得先查当前最大工号是多少，然后+1，太麻烦了
- 用 AUTO_INCREMENT？数据库自动帮你+1！

```sql
emp_id INT NOT NULL AUTO_INCREMENT COMMENT '工号'
```

**效果**：
| 操作 | 自动生成的emp_id |
|------|------------------|
| 插入第1个员工 | 1001（因为设了起始值） |
| 插入第2个员工 | 1002 |
| 插入第3个员工 | 1003 |

### 3.2 起始值怎么设？

```sql
)ENGINE=InnoDB AUTO_INCREMENT=1001 DEFAULT CHARSET=utf8mb4
```

`AUTO_INCREMENT=1001` 表示第一个自动生成的值从1001开始，而不是默认的1。

> **为什么从1001开始？** 因为工号从1001开始看起来更正式，像"员工编号"，而不是1、2、3。

### 3.3 你项目中哪些表用了自增？

| 表 | 自增字段 | 起始值 |
|----|----------|--------|
| employee | emp_id | **1001**（显式设置） |
| project | proj_id | 默认1 |
| performance | review_id | 默认1 |
| attendance | att_id | 默认1 |
| leave_req | leave_id | 默认1 |
| overtime | ot_id | 默认1 |
| pos_change | change_id | 默认1 |
| salary_record | salary_id | 默认1 |
| bonus | bonus_id | 默认1 |
| deduction | deduct_id | 默认1 |

> **怎么看出起始值是1001还是1？**
> 看建表语句末尾的 `AUTO_INCREMENT=数字`：
> - 有 `AUTO_INCREMENT=1001` → 从1001开始（只有 employee 表）
> - 没有这行 → 默认从1开始（其他9张表）

**对比一下两张表的末尾写法**：

```sql
-- ✅ employee 表：显式设了起始值
)ENGINE=InnoDB AUTO_INCREMENT=1001 DEFAULT CHARSET=utf8mb4 COMMENT='员工表';

-- ✅ 其他表：没设起始值，走默认
)ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='项目表';
```

**运行结果**：
| 插入的第N条记录 | employee.emp_id | project.proj_id | attendance.att_id |
|----------------|-----------------|------------------|-------------------|
| 第1条 | 1001 | 1 | 1 |
| 第2条 | 1002 | 2 | 2 |
| 第3条 | 1003 | 3 | 3 |
| ... | ... | ... | ... |

**为什么 employee 起始值要设成1001？** 主要是：
1. **看着专业** — 工号1001、1002 看起来像"正式员工编号"，比1、2、3更有员工感
2. **便于管理** — 真实企业中工号一般从某个数字开始（便于按编号段区分部门/入职年份）
3. **避免歧义** — 工号1和"ID=1"看起来很像，容易混淆；工号1001一眼就能看出是员工

### 3.4 哪些表**没有**用自增？

| 表 | 主键 | 为什么不用自增？ |
|----|------|------------------|
| department | dept_code (VARCHAR) | 部门代码是人为定义的，如"D001" |
| position | pos_id (VARCHAR) | 职位ID是人为定义的，如"P001" |
| payment_method | emp_id (INT) | 主键就是员工工号，1:1绑定 |
| project_member | (emp_id, proj_id) | 联合主键，不适合自增 |
| sal_bonus_rel | (salary_id, bonus_id) | 联合主键，不适合自增 |
| sal_deduct_rel | (salary_id, deduct_id) | 联合主键，不适合自增 |

> **规律**：自增适合"不需要人为指定"的数字ID；如果ID有业务含义（如部门代码D001），或者主键来自其他表（如emp_id），就不用自增。

## 四、插入数据时的写法

### 有 AUTO_INCREMENT 的表

```sql
-- 不需要手动指定 emp_id，数据库自动生成
INSERT INTO employee (emp_name, gender, hire_date, emp_status, pos_id, dept_code)
VALUES ('张三', '男', '2024-01-15', '在职', 'P001', 'D001');
-- emp_id 会自动变成 1001
```

### 没有 AUTO_INCREMENT 的表

```sql
-- 必须手动指定 dept_code
INSERT INTO department (dept_code, dept_name)
VALUES ('D001', '技术部');
```

## 五、主键 vs 其他唯一标识

你可能会问：部门名称也不能重复吧？为什么不用部门名称做主键？

| 对比 | 主键（dept_code） | 部门名称（dept_name） |
|------|-------------------|----------------------|
| 会变吗？ | 不会变 | 可能改名字 |
| 长度 | 短，固定 | 长，可变 |
| 其他表引用 | 引用dept_code，简单 | 引用dept_name，复杂 |

> **设计原则**：主键应该是**不变的、简短的、无业务含义的**。这样即使业务变化（如部门改名），主键不需要跟着变，其他引用这个主键的表也不受影响。

## 六、本课小结

```
主键 PRIMARY KEY
├── 作用：唯一标识每一行
├── 规则：不能重复、不能为空
├── 单列主键：一个字段做主键（如 emp_id）
└── 联合主键：多个字段组合做主键（如 emp_id + proj_id）

自增 AUTO_INCREMENT
├── 作用：数据库自动生成递增的数字
├── 适用：不需要人为指定的数字ID
├── 不适用：有业务含义的ID、联合主键
└── 起始值：AUTO_INCREMENT=1001 从1001开始
```

**核心记忆**：
1. 主键 = 身份证号，唯一且不为空
2. 联合主键 = 多个字段组合起来才能唯一标识
3. AUTO_INCREMENT 让数据库自动编号，省心省力
