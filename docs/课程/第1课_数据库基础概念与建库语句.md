# 第1课：数据库基础概念与建库语句

---

## 一、什么是数据库？

想象一个图书馆：
- **图书馆** = 数据库（Database）：存放数据的大仓库
- **书架** = 表（Table）：按类别分好的数据存放区
- **一本书** = 一条记录（Row）：一条具体的数据
- **书的各个属性**（书名、作者、ISBN）= 字段（Column）：记录中的各个信息项

> **一句话理解**：数据库就是有组织地存储数据的仓库，SQL就是你和这个仓库"对话"的语言。

## 三、建库语句逐行解读

```sql
CREATE DATABASE IF NOT EXISTS salary_management
DEFAULT CHARACTER SET utf8mb4
COLLATE utf8mb4_general_ci;
```

### 3.1 `CREATE DATABASE IF NOT EXISTS`

| 关键词 | 含义 |
|--------|------|
| `CREATE DATABASE` | 创建一个新数据库 |
| `IF NOT EXISTS` | 如果数据库已经存在，就不报错，跳过创建 |

> **为什么要加 IF NOT EXISTS？**
> 想象你写了一个脚本，第一次运行时创建数据库，第二次运行时数据库已经存在了。如果不加这个，第二次运行就会报错。加了之后，就像在说"如果没有就建，有了就算了"。

### 3.2 `DEFAULT CHARACTER SET utf8mb4`

**字符集**决定了数据库能存什么文字。

| 字符集 | 能存什么 | 举例 |
|--------|----------|------|
| latin1 | 只能存西欧文字 | 英文、法文 ✅ 中文 ❌ |
| utf8 | 存大部分文字，但每个字符最多3字节 | 中文 ✅ emoji ❌ |
| **utf8mb4** | 最全，每个字符最多4字节 | 中文 ✅ emoji ✅ 罕见字 ✅ |

> **mb4 = most bytes 4**，就是"最多用4个字节"的意思。在MySQL中，`utf8mb4` 才是真正的UTF-8，`utf8` 是个"残缺版"（最多3字节）。**永远推荐用 utf8mb4**。

### 3.3 `COLLATE utf8mb4_general_ci`

**排序规则**决定了文字怎么比较和排序。

| 排序规则 | 特点 | 举例 |
|----------|------|------|
| `utf8mb4_general_ci` | 不区分大小写（ci = case insensitive） | "ABC" = "abc" |
| `utf8mb4_bin` | 区分大小写，按二进制比较 | "ABC" ≠ "abc" |

> **ci = Case Insensitive**（不区分大小写），所以用 `general_ci` 时，查询 `WHERE name = 'zhang'` 也能匹配到 `Zhang`。

## 四、USE 语句

```sql
USE salary_management;
```

这句话的意思是：**"接下来我操作的都在这个数据库里"**。

就像你走进图书馆后说"我要在二楼的书架找书"，之后的操作都默认在这个数据库里进行。

## 五、建表语句的基本骨架

以部门表为例：

```sql
CREATE TABLE department(
    dept_code VARCHAR(10) NOT NULL COMMENT '部门代码',
    dept_name VARCHAR(50) NOT NULL COMMENT '部门名称',
    parent_dept_code VARCHAR(10) DEFAULT NULL COMMENT '上级部门代码',
    dept_manager_emp INT DEFAULT NULL COMMENT '部门负责人工号',
    PRIMARY KEY(dept_code)
)ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='部门表';
```

**骨架结构**：

```
CREATE TABLE 表名(
    字段1  数据类型  约束  COMMENT '注释',
    字段2  数据类型  约束  COMMENT '注释',
    ...
    PRIMARY KEY(主键字段)
)ENGINE=存储引擎 DEFAULT CHARSET=字符集 COMMENT='表注释';
```

| 组成部分 | 作用 | 类比 |
|----------|------|------|
| 字段名 | 给这一列起个名字 | 表格的列标题 |
| 数据类型 | 这一列能存什么数据 | 这一列是填数字还是填文字 |
| 约束 | 对数据加规则 | "这一列必填"、"这一列不能重复" |
| COMMENT | 注释，说明这个字段是什么意思 | 说明书 |
| PRIMARY KEY | 主键，唯一标识每一行 | 身份证号 |
| ENGINE | 存储引擎，决定数据怎么存 | 文件柜的类型 |
| CHARSET | 字符集 | 用什么编码存文字 |

## 六、本课小结

```
数据库 (Database)
  └── 表 (Table) - 用 CREATE TABLE 创建
        ├── 字段 (Column) - 定义数据类型和约束
        ├── 主键 (Primary Key) - 唯一标识每一行
        └── 注释 (Comment) - 帮助理解
```

**你学到了**：
1. 数据库 = 有组织的数据仓库
2. `CREATE DATABASE IF NOT EXISTS` 建库，`USE` 选库
3. `utf8mb4` 是最推荐的字符集，`general_ci` 不区分大小写
4. 建表的基本骨架：字段 + 数据类型 + 约束 + 主键
